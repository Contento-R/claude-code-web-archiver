# Claude Code Web Session Archiver

> Сохраняет **всю** сессию Claude Code Web — включая историю, которую облако уже сжало, — в один самодостаточный HTML-файл со встроенными скриншотами в виде data-URL.

Небольшой userscript для Tampermonkey / Violentmonkey / Greasemonkey, который добавляет кнопку **⬇ Архивировать сессию** на [claude.ai/code](https://claude.ai/code) и [claude.com/code](https://claude.com/code). По клику он:

1. Прокручивает чат в самое начало.
2. Постепенно прокручивает вниз, давая виртуализатору отрисовать каждое сообщение.
3. Разворачивает все свёрнутые блоки `<details>` и `aria-expanded="false"` в области видимости.
4. Захватывает все сообщения в хронологическом порядке (с дедупликацией по содержимому).
5. Скачивает все скриншоты из переписки и **встраивает их в виде `data:` URL**.
6. Собирает один портативный HTML-файл (тёмная тема, адаптивная вёрстка) и предлагает скачать.

Итоговый файл полностью автономен: ни одного внешнего запроса, ни одной потерянной картинки, открывается в любом браузере.

## Зачем

Claude Code Web сжимает длинные сессии. Старые сообщения исчезают из живой ленты и остаются только в виде краткого пересказа. Этот скрипт обходит живой DOM, пока всё ещё отрисовано, и сохраняет полный транскрипт (со скриншотами) до следующего сжатия.

## Установка

1. Установите менеджер userscript-ов:
   - [Tampermonkey](https://www.tampermonkey.net/) (Chrome, Edge, Firefox, Safari, Opera)
   - [Violentmonkey](https://violentmonkey.github.io/) (Chromium / Firefox)
   - [Greasemonkey](https://www.greasespot.net/) (Firefox)
2. Откройте сырой файл скрипта — менеджер сам подхватит его:
   - **Русская сборка:** [`i18n/claude-code-web-archiver.ru.user.js`](https://github.com/Contento-R/claude-code-web-archiver/raw/main/i18n/claude-code-web-archiver.ru.user.js)
   - **Английская (по умолчанию):** [`claude-code-web-archiver.user.js`](https://github.com/Contento-R/claude-code-web-archiver/raw/main/claude-code-web-archiver.user.js)
3. Подтвердите установку в менеджере.
4. Откройте любую сессию по адресу `https://claude.ai/code/*` или `https://claude.com/code/*`. В правом нижнем углу появится синяя кнопка **⬇ Архивировать сессию**.

## Использование

1. Откройте нужную сессию Claude Code Web.
2. Нажмите **⬇ Архивировать сессию**.
3. Подтвердите запрос. **Не трогайте страницу** во время работы — взаимодействие сбивает авто-прокрутку и захват.
4. Следите за прогрессом в плашке:
   - `Прокрутка и захват… сообщений: N`
   - `Скачиваю скриншоты… X/Y (Z ok)`
   - `Собираю HTML…`
   - `Готово! Сообщений: N, картинок встроено: M.`
5. Браузер сохранит файл `<название-сессии>.html`.

Чтобы прервать процесс, нажмите **Отмена** в плашке прогресса.

## Формат вывода

- Один `.html` файл, полностью самодостаточный, без внешних зависимостей.
- Тёмная тема, читается и на десктопе, и на мобильном.
- Каждое сообщение — `<section class="msg user|assistant">` с ролью и номером.
- Картинки встроены как `data:` URL, файл работает офлайн.
- Сохраняются блоки кода, таблицы, списки и раскрывающиеся `<details>`.
- В шапке: время архивации, число сообщений, исходный URL и версия парсера.

## Настройки

Поведение управляется объектом `CFG` в начале скрипта:

| Ключ               | По умолчанию | Что значит                                                  |
| ------------------ | -----------: | ------------------------------------------------------------ |
| `scrollStepRatio`  |        `0.6` | Доля высоты viewport на один шаг прокрутки                   |
| `scrollWaitMs`     |        `650` | Пауза после каждой прокрутки (мс) — увеличьте на медленном сети |
| `expandWaitMs`     |        `120` | Пауза после разворачивания блока                            |
| `stableLimit`      |          `4` | Сколько шагов без прогресса до остановки                    |
| `maxSteps`         |       `4000` | Жёсткий предохранитель по числу шагов                       |
| `minTextLen`       |          `8` | Игнорировать узлы короче этой длины (пустые обёртки)         |
| `imgTimeoutMs`     |      `20000` | Таймаут на скачивание одной картинки (мс)                   |

Меняйте прямо в установленном скрипте — менеджер применит изменения сразу.

## Права / сеть

Скрипт запрашивает `GM_xmlhttpRequest` и `@connect` к:

- `claude.ai`, `claude.com`, `anthropic.com` — контейнеры сообщений и вложения
- `cloudfront.net`, `amazonaws.com` — CDN для картинок
- `self` — same-origin запросы

Никаких данных никуда не отправляется. Всё происходит локально в браузере, файл вы сохраняете сами.

## Ограничения

- Скрипт полагается на структуру DOM, которую отдаёт Claude Code Web. Серьёзные изменения UI могут временно сломать селекторы. PR-ы приветствуются.
- Очень длинные сессии могут занять несколько минут на прокрутку, разворачивание и скачивание скриншотов — наберитесь терпения и не трогайте вкладку.
- Определение роли (user/assistant) — эвристика, на сильно кастомных сообщениях может ошибаться.
- Вложения, которые не являются `<img>` (например, скачиваемые файлы), не встраиваются.
