# Claude Code Web Session Archiver

> Сохраняет **всю** сессию Claude Code Web — включая историю, которую облако уже сжало, — в один самодостаточный HTML-файл со встроенными скриншотами в виде data-URL.

Небольшой userscript для Tampermonkey / Violentmonkey / Greasemonkey, который добавляет маленькую перетаскиваемую зелёную панель на [code.claude.com](https://code.claude.com), [claude.ai/code](https://claude.ai/code) и [claude.com/code](https://claude.com/code). На панели три элемента управления:

| Кнопка          | Что делает                                                                                                          |
| --------------- | ------------------------------------------------------------------------------------------------------------------- |
| ⬇ **Архив**     | Обходит всю сессию, захватывает каждое сообщение, скачивает скриншоты и сохраняет один автономный HTML.            |
| ⚡ **Быстро**    | Переключатель. Минимизирует задержки и распараллеливает загрузку картинок. Чуть агрессивнее к странице.            |
| 📝 **Без кода** | Переключатель. Исключает блоки кода, которые пишет Claude (tool-вызовы / `<pre>`). Выгружается только переписка.   |

Панель перетаскивается за «уголок» — небольшую вертикальную полоску слева. Позиция запоминается между визитами.

UI автоматически выбирает **английские** или **русские** строки по `navigator.language`.

## Зачем

Claude Code Web сжимает длинные сессии. Старые сообщения исчезают из живой ленты и остаются только в виде краткого пересказа. Этот скрипт обходит живой DOM, пока всё ещё отрисовано, и сохраняет полный транскрипт (со скриншотами) до следующего сжатия.

## Установка

1. Установите менеджер userscript-ов:
   - [Tampermonkey](https://www.tampermonkey.net/) (Chrome, Edge, Firefox, Safari, Opera)
   - [Violentmonkey](https://violentmonkey.github.io/) (Chromium / Firefox)
   - [Greasemonkey](https://www.greasespot.net/) (Firefox)
2. Откройте сырой файл скрипта — менеджер сам подхватит его:
   [`claude-code-web-archiver.user.js`](https://github.com/Contento-R/claude-code-web-archiver/raw/main/claude-code-web-archiver.user.js)
3. Подтвердите установку.
4. Откройте любую сессию по адресу `https://code.claude.com/*`, `https://claude.ai/code/*` или `https://claude.com/code/*`. В правом нижнем углу появится зелёная панель.

## Использование

1. Откройте нужную сессию Claude Code Web.
2. (Опционально) Включите **⚡ Быстро** для ускоренного захвата и/или **📝 Без кода** для пропуска блоков кода.
3. Нажмите **⬇ Архив** и подтвердите запрос.
4. **Не трогайте страницу**, пока скрипт работает — взаимодействие сбивает авто-прокрутку и захват.
5. Следите за прогрессом в плашке:
   - `Прокрутка и захват… сообщений: N`
   - `Скачиваю скриншоты… X/Y (Z ok)`
   - `Собираю HTML…`
   - `Готово! Сообщений: N, картинок встроено: M.`
6. Браузер сохранит файл `<название-сессии>.html`.

Чтобы прервать процесс, нажмите **Отмена** в плашке прогресса.

### Режим ускорения (⚡)

Включённый режим переключает скрипт на агрессивный профиль таймингов и параллелит загрузку картинок:

|                          | Обычный | Быстрый |
| ------------------------ | ------: | ------: |
| Пауза прокрутки (мс)     |     650 |     160 |
| Пауза разворачивания (мс)|     120 |      25 |
| Шаг прокрутки (vh)       |    0.6  |     0.9 |
| Шагов без прогресса      |       4 |       3 |
| Параллельных загрузок    |       4 |       8 |

Рекомендую на быстром железе и хорошей сети. На медленном — оставьте выключенным, странице нужно время на отрисовку виртуализированных сообщений.

### Режим «Без кода» (📝)

При включении архиватор удаляет из каждого захватываемого сообщения `<pre>`-блоки и контейнеры с `data-language` / `code-block` ещё до сохранения. Инлайновый `code` внутри текста остаётся. Переключайте **до** нажатия «Архив» — настройка действует на сообщения, захваченные после включения.

### Перетаскивание панели

Возьмитесь за полоску `⋮⋮` на левой стороне панели и тащите куда угодно. Позиция сохраняется в `localStorage` (`cc-arch-pos`) и восстанавливается при следующем открытии.

## Формат вывода

- Один `.html` файл, полностью самодостаточный, без внешних зависимостей.
- Тёмная тема, читается и на десктопе, и на мобильном.
- Каждое сообщение — `<section class="msg user|assistant">` с ролью и номером.
- Картинки встроены как `data:` URL, файл работает офлайн.
- Сохраняются блоки кода, таблицы, списки и раскрывающиеся `<details>` (если «Без кода» не включён).
- В шапке: время архивации, число сообщений, исходный URL и версия парсера.
- `<html lang>` — `en` или `ru` в зависимости от локали браузера.

## Настройки

Поведение управляется объектами `CFG_NORMAL` и `CFG_FAST` в начале скрипта:

| Ключ               | Обычный | Быстрый | Что значит                                                  |
| ------------------ | ------: | ------: | ------------------------------------------------------------ |
| `scrollStepRatio`  |    0.6  |     0.9 | Доля высоты viewport на один шаг прокрутки                   |
| `scrollWaitMs`     |    650  |     160 | Пауза после каждой прокрутки (мс)                            |
| `expandWaitMs`     |    120  |      25 | Пауза после клика по `aria-expanded`                         |
| `stableLimit`      |      4  |       3 | Сколько шагов без прогресса до остановки                    |
| `maxSteps`         |   4000  |    4000 | Жёсткий предохранитель по числу шагов                       |
| `minTextLen`       |      8  |       8 | Игнорировать узлы короче этой длины                          |
| `imgTimeoutMs`     |  20000  |   20000 | Таймаут на скачивание одной картинки (мс)                   |
| `concurrency`      |      4  |       8 | Параллельных загрузок скриншотов                            |

Меняйте прямо в установленном скрипте — менеджер применит изменения сразу.

## Права / сеть

Скрипт запрашивает `GM_xmlhttpRequest` и `@connect` к:

- `code.claude.com`, `claude.ai`, `claude.com`, `anthropic.com` — контейнеры сообщений и вложения
- `cloudfront.net`, `amazonaws.com` — CDN для картинок
- `self` — same-origin запросы

Никаких данных никуда не отправляется. Всё происходит локально в браузере, файл вы сохраняете сами.

## Ограничения

- Скрипт полагается на DOM, который отдаёт Claude Code Web. Серьёзные изменения UI могут временно сломать селекторы. PR-ы приветствуются.
- Очень длинные сессии могут занять несколько минут — наберитесь терпения и не трогайте вкладку.
- Определение роли (user/assistant) — эвристика, на сильно кастомных сообщениях может ошибаться.
- Вложения, которые не являются `<img>` (например, скачиваемые файлы), не встраиваются.
- «Без кода» работает по эвристике (`<pre>`, `[class*=code-block]`, `[data-language]`). Редкие контейнеры могут проскочить.
